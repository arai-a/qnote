Read about: excessive permissions QNote extension development was started after XNote++ support was dropped. It aims to be XNote++ drop-in replacement. Please report any issues and/or ideas to support e-mail or GitHub (please do not use reviews section for that). In order to improve quality of the QNote extension I'm looking for beta testers. Please send request to support e-mail if you would like to participate.Thank you! Features:
  • Add notes to messages
  • Save note position, size
  • Column with note icon and/or text preview
  • Note attaching to message
  • Note printing together with message
  • Fully XNote++ compatible
Usage:
  • Use right click context menu to delete note or reset note window
  • Use ESC to close note without saving
  • Use Alt+Q to toggle note
If you are using internal storage (not folder) then don't forget to export data before removing extension ! History:
  • Version 0.10: configurable localized date format, set default note placement, optional confirm delete, save options w/o extension reload, options page styling according to selected theme
  • Version 0.9: attaching note to message and printer, major rewrite
  • Version 0.8: non published
  • Version 0.7: note search feature using QuickFilter
  • Version 0.6: save notes using JSON file format instead of dated XNote++ format
  • Version 0.5: add QNote column and two options for popup window style - floating panel (preferred) or windows using TB new API
  • Version 0.4: two storage options - internal extension or XNote++ local folder (compatible with original extension)
Know issues:
  • Search is broken and currently disabled and will come back once a better solution is found
  • Does not work very well together with Conversations plugin because it does some heavy modifications to TB internals
  • 91.x version does not work well on Macs
